- **작성일**: 2026-07-14
- **작성자**: 안전진단팀
- **상태**: 검토 중 (Draft)

## 1. 개요 (Overview)

### 1.1 배경 및 문제 정의

안전진단팀은 현장 점검 일정과 체크리스트 진행 상황을 엑셀과 메신저로 나누어 관리하고 있다.

- 어느 현장이 점검 예정인지 여러 곳에 흩어져 있어 한눈에 파악이 안 됨
- 체크리스트를 어디까지 진행했는지 실시간으로 공유되지 않음
- 담당자·일정 변경 시 엑셀과 메신저를 각각 갱신해야 해 누락 위험 존재

### 1.2 제품 목적

| 목표 | 내용 |
|---|---|
| 가시성 확보 | 진행 중인 모든 점검 건을 카드 형태로 한 화면에서 확인 |
| 진행 상황 자동화 | 체크리스트 완료 항목에 따라 진행률과 상태 배지를 자동 계산 |
| 단순함 유지 | 비개발자도 즉시 사용할 수 있는 입력·표시 UI |

### 1.3 제품 범위

- **MVP 범위**: 점검 건 입력, 카드 목록 표시, 체크리스트 진행률 계산, 상태 배지 자동 전환, 카드 수정·삭제
- **제외 범위(추후 검토)**: 로그인/회원가입, 데이터베이스 저장(새로고침 시 초기화됨), 알림/메신저 연동, 다중 사용자 동시 편집, 엑셀 내보내기

## 3. 핵심 기능 (MVP)

### 3.1 점검 건 입력

입력 폼에서 아래 항목을 받아 새 카드를 생성한다.

| 항목 | 설명 | 필수 여부 |
|---|---|---|
| 현장명 | 텍스트 입력 | 필수 |
| 점검 예정일 | 날짜 선택 | 필수 |
| 담당자 | 텍스트 입력 | 필수 |
| 점검 항목 체크리스트 | 기본 4종(안전난간, 소화기, 전기설비, 대피로) 자동 포함, 자유 추가/삭제 가능 | 최소 1개 이상 |

### 3.2 카드 목록 표시

- 등록된 점검 건은 화면 상단(또는 목록 영역)에 카드 형태로 쌓인다.
- 각 카드에는 현장명, 점검 예정일, 담당자, 체크리스트, 진행률, 상태 배지가 표시된다.

### 3.3 체크리스트 & 진행률

- 카드 내 체크리스트 항목을 체크/해제하면 진행률이 즉시 재계산된다.
- 진행률 = (체크된 항목 수 / 전체 항목 수) × 100, 예: 4개 중 2개 체크 = 50%

### 3.4 상태 배지 자동 전환

체크리스트 진행률만을 기준으로 자동 전환한다.

| 진행률 | 상태 배지 |
|---|---|
| 0% | 예정 |
| 1% ~ 99% | 진행중 |
| 100% | 완료 |

### 3.5 카드 수정·삭제

- 현장명, 점검 예정일, 담당자, 체크리스트 항목(추가/삭제/이름 변경)을 카드 단위로 수정할 수 있다.
- 카드 전체를 삭제할 수 있다(확인 절차 포함).

## 4. 비기능 요구사항

- **데이터 저장**: 컴포넌트 state로만 관리, 별도 DB 없음 → 새로고침 시 데이터 초기화됨(허용됨)
- **인증**: 로그인/회원가입 기능 없음
- **디자인**: Notion 스타일의 깔끔한 업무용 웹 디자인, 카드형 레이아웃
- **테마**: 다크/라이트 모드 모두 지원
- **기술 스택**: React / Next.js

---

## 5. 제외 사항 (Out of Scope)

- 로그인·회원가입, 권한 관리
- 서버/DB 영속 저장
- 알림, 메신저 연동
- 엑셀 내보내기, 인쇄용 출력
- 다중 사용자 동시 편집 및 실시간 동기화

---

## 6. 향후 고려 사항 (Post-MVP)

- 데이터베이스 연동을 통한 영속 저장
- 현장/담당자별 필터링, 검색, 정렬
- 점검 예정일 기준 지연 상태(예: "지연") 추가
- 엑셀/PDF 내보내기
- 알림(메신저 연동) 기능
